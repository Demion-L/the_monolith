# Vision

Long-term direction for MONOLITH.

Vision documents describe where the system is going, not what the system is today.
They are not commitments. They are not timelines. They are direction.

For the current system, see `docs/architecture/`.
For the decisions that shaped the current system, see `docs/adr/`.
For the research that informed those decisions, see `docs/research/`.

## Vision Principles

- Vision documents do not define implementation.
- Vision documents do not supersede architecture documents.
- When a vision concept is ready to implement, it first becomes a research document (R-XXXX), then an ADR, then an architecture document.
- Vision documents are updated when direction changes, not when implementation progresses.
