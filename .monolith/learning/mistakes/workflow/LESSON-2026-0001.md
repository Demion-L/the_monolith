# Lesson: Check CLAUDE.md for skill triggers before acting

## Mistake

User said "финализируй 1.3". I proceeded directly to implementation (created ADR, committed code) without checking whether a MONOLITH skill applied to this request. The `finalize-story` skill existed and was the correct procedure, but I didn't look for it.

## Root Cause

No activation mechanism forced me to read `.monolith/skills/` before acting on trigger phrases. CLAUDE.md did not exist. The skill index was a passive document, not an enforced gate.

## Prevention

**Before starting any task:** Read CLAUDE.md skill activation table. If the user's request matches a trigger phrase, read the corresponding SKILL.md before taking any action.

**Signal to watch for:** User uses a verb that maps to a known workflow — "finalize", "status", "analyze mistake", "финализируй", "статус". Stop. Check CLAUDE.md first.

## Criteria check

- Recurs in other stories: yes — any story can be finalized, any session can start with a status check
- Non-obvious cause: yes — no enforcement existed before CLAUDE.md was created
- Concrete prevention: yes — CLAUDE.md now exists; read it at session start and on trigger phrases
