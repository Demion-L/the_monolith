# Lesson: Register a story in active.md before starting work

## Mistake

Story 1.3 was started and executed across three sub-stories without ever updating `memory/working/active.md`. The file showed "No active story" throughout, making the `finalize-story` skill's pre-condition check meaningless and leaving working memory out of sync with actual project state.

## Root Cause

No step in the story-start workflow required updating active.md. Story start was implicit — the user described the story in conversation, and work began immediately without a registration step.

## Prevention

**Before starting work on a new story:** Update `memory/working/active.md` — set the active story name, objective, and phase. This takes one edit and costs nothing.

**Signal to watch for:** User defines a new story or sub-story objective. Before writing any code or docs: update active.md first.

## Criteria check

- Recurs in other stories: yes — every story starts; this omission can happen at the beginning of any story
- Non-obvious cause: yes — there's no existing skill or protocol for story-start; the gap is structural
- Concrete prevention: yes — one specific file to update, one specific moment (before first implementation action)
